What I Learned So Far (Beginner Version)
==================================================

1) Package structure
- I created a dedicated user feature area under:
  backend/src/main/java/com/lifeforest/backend/user
- This keeps user-related code grouped together and easier to understand.

2) Enum (UserRole)
- File: user/domain/UserRole.java
- Purpose: define allowed user roles (USER, ADMIN).
- Why: avoid random strings and typos for role values.

3) Entity (User)
- File: user/domain/User.java
- Purpose: blueprint for how a user is stored in the database.
- Includes: id, email, passwordHash, displayName, role, createdAt, updatedAt.
- Why: this is the core database model.

4) Repository (UserRepository)
- File: user/repository/UserRepository.java
- Purpose: read and write User data using Spring Data JPA.
- Methods added: findByEmail, existsByEmail.
- Why: needed to prevent duplicate emails and fetch users.

5) Request DTOs
- Files:
  user/dto/request/UserCreateRequestDto.java
  user/dto/request/UserUpdateRequestDto.java
- Purpose: define what API input should look like.
- Why: protect app from bad input and keep validation rules clear.

6) Response DTO
- File: user/dto/response/UserResponseDto.java
- Purpose: define what API sends back.
- Why: send only safe fields (never send password or passwordHash).

7) Mapper (UserMapper)
- File: user/mapper/UserMapper.java
- Purpose: translate between DTOs and Entity.
- Why: keeps conversion logic in one place and avoids duplicated code.

8) Build/test checks
- compileJava: checks if source code compiles.
- test: runs tests.
- Note: tests originally failed due to missing test datasource; fixed by adding H2 test setup.
